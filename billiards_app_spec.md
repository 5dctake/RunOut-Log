## 1. プロジェクト概要

### 1.1 アプリの目的

ビリヤードの取り切り練習（3・4・5個、将来的に6〜9個）を記録・分析するスポーツ練習管理ツール。単なる記録にとどまらず、**成長の可視化**と**モチベーション維持**を中心設計思想とする。

- **成長の可視化**：100回単位のブロックで長期的な成功率の変遷を確認
- **練習量の可視化**：累計試行数を常に表示し、練習量を数値で実感
- **成功率の分析**：セット単位・ブロック単位・累計の多角的な分析
- **モチベーション維持**：スポーツ寄りのダークデザインで高い継続意欲

### 1.2 ターゲットユーザー

ビリヤード（ポケット・キャロム問わず）の取り切りを定期的に練習するプレイヤー。初心者から中上級者まで幅広く対応。

### 1.3 技術スタック

| 項目 | 内容・選定理由 |
|------|---------------|
| フレームワーク | Flutter（iOS / Android クロスプラットフォーム対応） |
| ローカルDB | Hive（高速・軽量 NoSQL、Flutter ネイティブ対応） |
| 将来のクラウド | Firebase Firestore（有料版での端末間同期） |
| 状態管理 | Riverpod または Provider（要検討） |
| グラフ描画 | fl_chart パッケージ |

## 2. 画面構成・仕様詳細

> アプリ全体は下部ナビゲーションバーで4画面に分かれる：**記録 | 入力 | グラフ | 設定**

### 2.1 記録画面（スコアボード）

練習記録をスコアボード形式で表示する、**閲覧専用**の画面。入力画面とは完全に分離。

#### 上部情報エリア

- **種目タブ**：3球 / 4球 / 5球 / …（将来拡張対応）
- **表示範囲ラベル**：`101–200` のように累計基準で表示
- **範囲内成功率**：その100回分の成功率（例：55%）
- **累計成功率・累計試行数**：全記録を通じた累計（例：61%（286回））

#### 中央スコアリスト

- 1ページ = 100試行（固定）
- 10行表示（1行 = 1セット10回）
- 各行の右端にセット成功率を表示
- 100回未満の最終ページは `201–286` のように動的表示

#### 表示例

```
101–200  |  成功率：55%  |  累計：61%（286回）
─────────────────────────────────────────────
02/17   ○ × ○ ○ × ○ ○ ○ × ○   70%
02/18   ○ ○ × × ○ ○ ○ × ○ ○   70%
02/19   × ○ × ○ ○ × ○ ○ ○ ×   60%
...（10行まで）
```

#### ナビゲーション

- 左右スワイプで100回ブロックを切り替え
- 画面端の左右ボタンでも切り替え可能

---

### 2.2 入力画面

記録画面とは完全に分離された**入力専用**画面。大きなグリッドで入力しやすいUIを提供。

#### 画面要素

- **日付表示**：自動で当日日付を表示（変更可能）
- **種目表示**：現在選択中の種目（例：5球取り切り）
- **5×2 グリッド**：10マスを2列に配置、各マスをタップで○/×切り替え
- **成功率リアルタイム表示**：タップのたびにその場で成功率を更新
- **保存ボタン**：タップで Hive に保存し記録画面へ反映

#### デザイン要件

- マスは最低 44×44dp 以上（タップしやすいサイズ）
- ○ = ライムグリーン `#52B788` 背景
- × = チャコール `#343A40` 背景
- 未選択 = グレー枠のみ

---

### 2.3 グラフ画面（分析）

練習データをグラフで可視化する分析専用画面。2種類のグラフをタブで切り替える。

#### 上部情報エリア

- 種目（タブ選択）
- 累計成功率・総試行数

#### グラフ①：100回推移グラフ（メイン）

`0–100`、`101–200`、`201–300` … のブロック単位で成功率を**折れ線グラフ**表示。長期的な成長トレンドを確認できる。

```
Y軸: 成功率 (0–100%)
X軸: 0–100 / 101–200 / 201–300 / ...
線色: ライムグリーン (#52B788)
背景: ダークグリーン (#1B4332)
```

#### グラフ②：セット推移グラフ

セット（10回）単位で成功率を表示。日ごとの波形・調子の変動を確認できる。

```
例: 70 → 60 → 80 → 50 → 70 → 80 ...
棒グラフ or 折れ線グラフ（どちらでも可）
X軸: セット番号 or 日付
```

#### タブ切り替え

```
[ 100回推移 ]  [ セット推移 ]
```

## 3. データ構造・ロジック仕様

### 3.1 PracticeRecord モデル

```dart
class PracticeRecord {
  DateTime date;       // 記録日付
  int ballCount;       // 種目：3, 4, 5, ... (将来拡張)
  List<bool> results;  // 10要素固定 (true=○, false=×)
}

// 派生プロパティ（計算）
int get successCount => results.where((r) => r).length;
double get successRate => successCount / results.length * 100;
```

### 3.2 100回ブロック方式

記録は累計試行数を基準に100回単位でブロック分けする。ブロックは連続しており、先頭は常に0からカウント。

```
ブロック 1: 試行   1–100
ブロック 2: 試行 101–200
ブロック 3: 試行 201–300
（最終ブロックが100回未満の場合は動的表示）
例: 286回の場合 → 最終ページは「201–286」と表示
```

### 3.3 集計ロジック

- **種目別に独立して集計**（3球の記録と5球の記録は別管理）
- **累計成功率** = 全試行の成功数 ÷ 全試行数 × 100
- **ブロック成功率** = そのブロック内の成功数 ÷ ブロック内試行数 × 100
- **セット成功率** = 1セット（10回）内の成功数 × 10（パーセント換算）

### 3.4 Hive ストレージ設計

```dart
// Box名: 'practice_records'
// Key: 自動インクリメント or UUID
// 種目フィルタリングはアプリ側で実施

// Hive アダプター必要
@HiveType(typeId: 0)
class PracticeRecord extends HiveObject { ... }
```

## 4. デザイン仕様

### 4.1 カラーパレット

| カラー名 | HEX | 用途 |
|---------|-----|------|
| ダークグリーン | `#1B4332` | 全画面の背景色 |
| ライムグリーン | `#52B788` | ○マーク・成功・強調 |
| チャコール | `#343A40` | ×マーク・失敗 |
| アクセントグリーン | `#40916C` | ヘッダー・見出し・ボーダー |
| ライトグリーン | `#F0FAF4` | カード背景・薄いエリア |
| テキスト（濃） | `#1A1A2E` | 本文テキスト |
| テキスト（薄） | `#ADB5BD` | 補助情報・下線 |
| 白 | `#FFFFFF` | 主要テキスト（ダーク背景上） |

### 4.2 タイポグラフィ

- フォント：システムデフォルト（日本語対応）
- 成功率表示：特大・太字（48sp以上）
- 累計試行数：中サイズ・太字（24sp）
- セット行テキスト：等幅フォント推奨（○×の並びを揃える）
- 数字は全て太字

### 4.3 UIコンポーネント方針

- **下部ナビゲーションバー**：4アイコン（記録 | 入力 | グラフ | 設定）
- **種目タブ**：上部タブバー（2〜5個、スクロール対応）
- **入力グリッド**：GestureDetector で大きなタップ領域を確保
- **グラフ**：fl_chart の LineChart / BarChart を使用
- **ダイアログ**：データ削除時は必ず確認ダイアログを表示

### 4.4 アニメーション・インタラクション

- 記録画面のスワイプ：PageView で滑らかに切り替え
- 入力マスのタップ：即時色変更 + 小さなスケールアニメーション
- 成功率の更新：AnimatedSwitcher でフェード更新
- グラフ描画：fl_chart のデフォルトアニメーション使用

## 8. 非機能要件・品質基準

### 8.1 パフォーマンス

- 記録画面の初期表示：500ms以内
- 入力マスのタップ応答：即時（16ms以内）
- グラフ描画：1000ms以内
- Hive 読み込み：バックグラウンドで非同期実行

### 8.2 対応環境

- iOS 13.0 以上
- Android 6.0 (API 23) 以上
- 画面サイズ：5.5〜6.7インチを主対象
- タブレット対応：Phase 2以降で検討

### 8.3 データ保護

- Hive は端末ローカルに保存（MVP段階は暗号化なし）
- データ削除は確認ダイアログ必須（誤削除防止）
- 将来の Firebase 移行時は認証を必須化
